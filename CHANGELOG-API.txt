## API Versioning Strategy

### Backend Versioning Guidelines
- **MAJOR**: Breaking API changes, database schema changes, authentication changes
- **MINOR**: New endpoints, enhanced features, backward-compatible changes
- **PATCH**: Bug fixes, performance improvements, security patches

### API Endpoints Overview

#### Authentication
- `POST /api/v1/auth/login` - User login
- `POST /api/v1/auth/register` - User registration
- `GET /api/v1/auth/me` - Get current user
- `POST /api/v1/auth/dev-login` - Development login

#### Users
- `GET /api/v1/users` - List users (admin)
- `GET /api/v1/users/:id` - Get user details
- `PUT /api/v1/users/:id` - Update user
- `DELETE /api/v1/users/:id` - Delete user

#### Trainers
- `GET /api/v1/trainers` - List trainers
- `GET /api/v1/trainers/:id` - Get trainer details
- `PUT /api/v1/trainers/:id` - Update trainer profile
- `POST /api/v1/trainers/content` - Upload content

#### Medical
- `GET /api/v1/medical/doctors` - List doctors
- `POST /api/v1/medical/consult` - Book consultation
- `GET /api/v1/medical/applications` - Medical applications

#### Subscriptions
- `GET /api/v1/subscriptions/plans` - Get plans
- `POST /api/v1/subscriptions` - Create subscription
- `POST /api/v1/payments/initiate` - Initiate payment

#### Content
- `GET /api/v1/content` - List content
- `POST /api/v1/content` - Upload content
- `GET /api/v1/content/:id` - Get content details
